## 📖 Sobre o Projeto

O **ConectaBus** é um aplicativo criado para integrar diferentes meios de transporte urbano em uma única plataforma, permitindo que os usuários encontrem rotas, acompanhem lotação em tempo real, realizem pagamentos digitais e recebam alertas importantes durante seus deslocamentos.

O protótipo foi desenvolvido em baixa fidelidade utilizando o **Miro**, com foco em usabilidade, rapidez de navegação e redução da carga cognitiva do usuário.

---

# 🎯 Objetivo da Atividade

Criar um protótipo de baixa fidelidade para um aplicativo de mobilidade urbana, priorizando:

- Navegação simples e intuitiva;
- Uso rápido em ambientes externos;
- Facilidade de interação com apenas uma mão;
- Organização clara das informações;
- Redução de erros durante o uso.

---

# 📱 Funcionalidades Desenvolvidas

O protótipo possui as seguintes telas principais:

1. **Tela de Busca de Rotas**
   - Campo para digitar o destino;
   - Sugestões de locais frequentes;
   - Acesso rápido ao histórico de trajetos.

2. **Resultado de Rotas**
   - Comparação entre trajetos;
   - Destaque para opções mais rápidas e mais baratas;
   - Tempo estimado de chegada.

3. **Mapa de Percurso**
   - Visualização do trajeto;
   - Ícones representando ônibus, metrô e bicicletas;
   - Acompanhamento do deslocamento.

4. **Carteira Digital**
   - Exibição do saldo disponível;
   - QR Code para pagamento de passagem;
   - Acesso rápido para validação na catraca.

5. **Detalhes do Veículo**
   - Informações de lotação;
   - Indicação de acessibilidade;
   - Recursos disponíveis no transporte.

---

# 🌎 Contexto de Uso

O aplicativo foi pensado para situações reais do cotidiano urbano, como:

- Usuários correndo para não perder ônibus ou metrô;
- Pessoas utilizando o celular em pé ou em movimento;
- Ambientes movimentados e com distrações;
- Usuários segurando mochila, sacolas ou utilizando apenas uma mão;
- Necessidade de tomada de decisão rápida durante o trajeto.

---

# 🧠 Decisões de UX

Durante a criação do protótipo, algumas decisões foram tomadas para melhorar a experiência do usuário:

- O tempo de chegada do transporte recebeu maior destaque visual por ser a informação mais importante em situações de pressa;
- Botões grandes foram utilizados para facilitar o toque com apenas um polegar;
- A navegação entre telas foi simplificada para reduzir a quantidade de cliques;
- Informações secundárias foram organizadas abaixo das informações principais para evitar excesso visual;
- Ícones simples e reconhecíveis foram utilizados para facilitar a identificação rápida das funções.

---

# ♿ Acessibilidade

O protótipo foi desenvolvido considerando acessibilidade e praticidade:

- Elementos clicáveis maiores para facilitar o uso por pessoas com mobilidade reduzida;
- Layout simples para diminuir confusão visual;
- Informações importantes destacadas para usuários em situação de pressa;
- Indicação de acessibilidade nos veículos, como rampas e ar-condicionado;
- Navegação intuitiva e objetiva.

---

# 🔄 Fluxo de Navegação

O fluxo principal do aplicativo funciona da seguinte maneira:

1. Usuário pesquisa um destino;
2. O sistema apresenta opções de rotas;
3. O usuário escolhe o trajeto desejado;
4. O mapa exibe o percurso em tempo real;
5. O usuário acessa a carteira digital para pagamento;
6. O aplicativo fornece informações do veículo e alertas durante a viagem.
